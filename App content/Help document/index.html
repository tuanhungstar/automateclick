üöÄ Getting Started

What is AutomateTask?

AutomateTask is a powerful visual automation platform designed to empower users to create, manage, and execute automated workflows (referred to as "bots") with minimal to no programming. By using a graphical, drag-and-drop style interface, you can build complex sequences of actions to automate repetitive tasks, data processing, application interactions, and much more.

Main Interface Layout

The application is logically divided into three primary areas to provide an intuitive and efficient user experience:

  • Left Panel: This is your main command center. It houses global action buttons for executing your bot, saving your work, managing steps, and accessing advanced tools.
  • Center Area: This is your primary workspace. It uses a tabbed layout to switch between different views of your project, such as the list of saved bots, the step-by-step execution flow, and the visual workflow diagram.
  • Right Panel: This is your resource panel. It contains the Module Browser for adding new functions to your bot and the Global Variable manager for handling data across your workflow.

üß© Bot Templates

Bot Templates are reusable blocks of steps that you can save and easily insert into any workflow. They are perfect for standardizing common procedures like logging in, exporting a report, or filling out a form.

Creating a Template

Function: You can turn any block of steps (a Group, Loop, or If/Else block) into a template directly from the üìã Execution Flow or üìà Workflow view.

  • How it works: In the üìã Execution Flow, click the Save Template button on the block's card. In the üìà Workflow view, right-click on a block node and select "Save As Template".
  • Naming: You will be prompted to give your template a name. You can either create a new name or choose an existing template to overwrite.
  • Usage Scenario: If you have a sequence of 5 steps for logging into a web application, group them together and save the group as a "WebApp Login" template.

Using a Template

Function: Once saved, templates appear in the üîß Module Browser on the right-hand panel, under the "Bot Templates" section. You can add them to your workflow just like any other step.

  • How it works: Double-click a template in the browser. You will be prompted to choose where to insert it in your current üìã Execution Flow.
  • Variable Mapping: If the template uses variables (e.g., `@username`, `@password`), a dialog will appear allowing you to map them. You can map a template variable to one of your existing global variables or create a new global variable on the fly.
  • Organization: You can right-click on "Bot Templates" or any subfolder to create new folders, allowing you to organize your templates by function or application. Templates can be dragged and dropped between folders.

Managing Templates

You can manage your templates directly from the üîß Module Browser.

  • View Documentation: Right-click a template to see if it has associated documentation. This is useful for complex templates shared among a team.
  • Delete Template: Right-click a template and select "Delete Template" to permanently remove it.
  • Drag & Drop: Click and drag a template from one folder to another to reorganize your library.

üîÑ Add Loop

Loops allow you to repeat a set of actions multiple times. This is fundamental for processing lists of data, waiting for a condition, or performing a task a specific number of times.

üîÑ Add Loop

Function: This button opens the "Configure Loop Group" dialog, allowing you to define the parameters for a new loop block. When confirmed, it inserts a `Loop Start` and `Loop End` block into the üìã Execution Flow.

  • How it works: After clicking the button, you first configure the loop's behavior in a dialog. Then, you select where in the execution flow you want to place the new loop block. Any steps you place between the `Loop Start` and `Loop End` markers will be repeated.
  • Usage Scenario: You need to process every row in an Excel file. You can create a loop that runs for the total number of rows, and inside the loop, you place steps to read data from a row, process it, and write it somewhere else.

Loop Configuration Dialog

When you add a loop, you will be presented with the following options:

  • Loop Name: An optional, descriptive name for your loop (e.g., "Process Excel Rows"). This name will appear in the workflow view for clarity.
  • Loop Count: You can define how many times the loop should run in two ways:
    • Fixed Number: Enter a specific number (e.g., `10`) for a fixed number of iterations.
    • Global Variable: Use a global variable (e.g., `@RowCount`) to control the loop dynamically. This is powerful for adapting to data of varying sizes.
  • Assign Current Iteration to Variable: This is an optional but highly useful feature. You can assign the current loop number (starting from 1) to a new or existing global variable. This is essential for tasks like accessing data from a specific row number in each iteration (e.g., read from `C@{CurrentRow}`).

Editing a Loop

You can change a loop's configuration at any time. Simply click the Edit button on the `Loop Start` card in the üìã Execution Flow or right-click the "Loop" node in the üìà Workflow view and select "Configure Parameters". The changes will automatically apply to the entire block.

‚ùì Add Conditional

Conditional blocks (also known as If/Else blocks) allow your bot to make decisions. Based on whether a condition is true or false, the bot can execute different sets of steps, making your automation dynamic and intelligent.

‚ùì Add Conditional

Function: This button opens the "Configure Conditional Block" dialog. When confirmed, it inserts an `If Start`, `Else`, and `If End` block into the üìã Execution Flow, creating two distinct branches for your logic.

  • How it works: You define a condition (e.g., `@Amount > 100`). Steps placed between `If Start` and `Else` will run only if the condition is true. Steps placed between `Else` and `If End` will run only if the condition is false.
  • Usage Scenario: An automation processes invoices. You can use a conditional block to check if an invoice amount is over a certain threshold. If true, it follows steps to get manager approval. If false, it proceeds with standard payment processing.

Conditional Configuration Dialog

When you add a conditional block, you will define the decision-making logic:

  • Block Name: An optional, descriptive name for your block (e.g., "Check Invoice Amount"). This name will appear in the workflow view for clarity.
  • Condition: This is the core of the block, consisting of three parts:
    • Left Operand: The first value in your comparison. This can be a hardcoded value (like `100`, `"Approved"`, `True`) or a dynamic value from a @Global Variable.
    • Operator: The comparison to perform, such as `==` (equals), `!=` (not equals), `>` (greater than), `in`, `is`, etc.
    • Right Operand: The second value in your comparison, which can also be a hardcoded value or a @Global Variable.
  • Example Condition: Check if a variable named `@Status` contains the word "Error".
    • Left Operand: `Error` (Hardcoded Value)
    • Operator: `in`
    • Right Operand: `@Status` (Global Variable)

Editing a Conditional Block

To change the condition, click the Edit button on the `If Start` card in the üìã Execution Flow or right-click the "IF" node in the üìà Workflow view and select "Configure Parameters".

‚ÜïÔ∏è Rearrange Steps

This powerful tool provides a dedicated interface for organizing, grouping, and deleting steps in your workflow. It is especially useful for managing large and complex bots where simple up/down movements are inefficient.

‚ÜïÔ∏è Rearrange Steps

Function: This button opens the "Rearrange Bot Steps" dialog, which presents your entire workflow as a flat, re-orderable list.

  • How it works: The dialog displays each step with its number and a summary of its action. From here, you can perform large-scale changes to your bot's structure. When you click "OK", the main üìã Execution Flow is updated to reflect your changes.
  • Usage Scenario: You've built a 50-step bot and realize a block of 10 steps from the end needs to be moved near the beginning. Instead of clicking the "up" arrow dozens of times, you can use this dialog to select and move the entire block in one action.

Rearrangement Methods

The dialog offers multiple ways to manipulate your workflow:

  • Drag & Drop: The most intuitive method. Simply click and drag a step (or a selection of steps) to a new position in the list. A line indicator will show you exactly where the steps will be dropped.
  • Multi-Select: You can select multiple steps at once using standard keyboard shortcuts:
    • Ctrl + Click to select individual, non-consecutive steps.
    • Shift + Click to select a continuous range of steps.
  • Precise Move Controls: For exact placement, select one or more steps and use the "Move Selected Steps" controls at the bottom. You can choose to move your selection "Before Step #" or "After Step #" and specify the target step number.

Other Actions in the Dialog

Beyond reordering, you can perform other critical edits:

  • üì¶ Group Selected: Select multiple steps and click this button to wrap them in a new `Group` block. You will be prompted to enter a name for the group.
  • üóëÔ∏è Delete Selected: Select one or more steps and click this button to permanently remove them from the workflow after confirmation. This is much faster than deleting steps one by one.

üîÑ Update App

This function allows you to update the application to the latest version directly from the source repository. It provides a guided process to ensure new features and bug fixes are applied correctly.

üîÑ Update App

Function: Initiates the application update process. The tool will first attempt to download the update automatically. If that fails (due to network restrictions, for example), it will guide you through a manual download and update procedure.

Automatic Update Process

  1. Click the üîÑ Update App button. The application will attempt to download the latest version as a ZIP file.
  2. Once downloaded, the "Update Application" dialog will appear, showing a tree view of all new and changed files.
  3. By default, all files are selected. You can uncheck any files you do not wish to update.
  4. Click the Update Selected Files button to copy the new files into your application directory, overwriting the old ones.

Manual Update Process

If the automatic download fails, you will be prompted to perform a manual update:

  1. A dialog will appear with a direct link to the update file. Click the link to download the ZIP file in your web browser.
  2. Once the download is complete, return to the application and click the Select File... button in the dialog.
  3. Navigate to your "Downloads" folder (or wherever you saved the file) and select the ZIP file you just downloaded.
  4. The application will then proceed to the "Update Application" dialog (step 2 of the automatic process), allowing you to select and apply the updates.

Important Notes

  • Restart Required: After the update is complete, you must close and restart the application for the changes to take effect.
  • User Data is Safe: The update process only affects the application's core files. Your saved bots, templates, images, and global variables will not be affected.
  • Permissions: Ensure you have write permissions for the application's installation folder, or the update will fail.
